Reading in the image files

There are two modes for reading in files:

  • A single file containing multiple images - default
  • Multiple images of single planes (Image stack)

After choosing the input mode you initiate the read by clicking on the open file/directory icon.

This icon allows the image file/directory to be chosen. If this icon is pressed after an image has been loaded it will terminate the analysis and save the tetramer data associated with that image and load whatever image is chosen next. This allows different images to be processed in the same instance of the program

Reading in a single multi-image dataset

In this case you use the browser icon to select a single image.

On selection, the system searches for the file {fn_}tetramer.dat and reads it in if it is present. If there is no tetramer file it will ask you for the image scaling. Entering and modifying the scaling is described here. Display of the image and any previously loaded tetramers occurs when 'Load & Display" is pressed.

Reading in an image stack

Image stacks are assumed to consist of multiple images of approximately the same size which are read in in numerical order. For this reason it is only necessary to use the browse icon to identify the directory which contains the files. It is assumed that this directory contains no TIFF images other those in the image stack. In this case, {fn_} is set to NULL and the files saved are named tetramer.dat and nnd.dat

Selecting the directory name automatically lists the files to be read in, in numerical order, and reads in any existing {fn_}tetramer.dat file, as seen in the image below.

If there is no tetramer file it will ask you for the image scaling if you haven't entered it previously. Entering and modifying the scaling is described here.

Pressing 'Load & display' will read in and display images and any previously saved tetramers. Adding, modifying, deleting and classification of the tetramers is described here

NOTE: An image which was originally analysed as a multi-image file which is then displayed as a single multilayer imagetack may be inverted compared to the original multi-image file. If the saved tetramers do not coincide with the image use the F12 key to flip the image and bring them back into register
